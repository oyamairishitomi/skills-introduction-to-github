# GitHub入門

_1時間以内にGitHubを使い始めよう。_

> **Note**
> このコースは [GitHub Skills: Introduction to GitHub](https://github.com/skills/introduction-to-github) の日本語版です。

## ようこそ

世界中の人々がGitHubを使って、最先端のテクノロジーを作っています。データの可視化でもゲーム開発でも、GitHubにはあなたをサポートするコミュニティとツールがあります。このGitHub Skills「GitHub入門」では、1時間以内にコントリビューション（貢献）を始めるために必要なことをすべて学べます。

- **対象者**: 新しい開発者、GitHubを初めて使う人、学生
- **学ぶこと**: リポジトリ、ブランチ、コミット、プルリクエストについて紹介します
- **作るもの**: [プロフィールREADME](https://docs.github.com/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme)として使える短いMarkdownファイルを作成します
- **前提条件**: なし。GitHubを初めて使う日にぴったりのコースです
- **所要時間**: 1時間以内で完了できます

このコースでは、以下のことを行います：

1. ブランチを作成する
2. ファイルをコミットする
3. プルリクエストを作成する
4. プルリクエストをマージする

### コースの始め方

コースを自分のアカウントにコピーし、Octocat（Mona）が最初のレッスンを準備するまで**約20秒**待ってから、**ページを更新**してください。

<summary>うまくいかない場合 🤷</summary><br/>

コースをコピーする際の推奨設定：

- オーナーには、自分の個人アカウントまたは組織を選択してください。

- パブリックリポジトリの作成をおすすめします。プライベートリポジトリではActionsの実行時間を消費します。

20秒経ってもコースが準備されない場合は、[Actions](../../actions) タブを確認してください。

- ジョブが実行中か確認してください。少し時間がかかることがあります。

- 失敗したジョブが表示された場合は、Issueを報告してください。バグを見つけました！ 🐛
